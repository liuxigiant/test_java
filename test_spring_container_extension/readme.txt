spring容器实例化bean的几种扩展处理方式：
1、配置初始化方法init-method
2、实现InitializingBean接口，类实例化后处理器
3、容器级别实例化后处理器BeanPostProcessor
4、定制类定义BeanFactoryPostProcessor


本工程实例，旨在熟悉以上四种处理方式，并验证各种处理的执行时间（类加载前、后  或者 实例化前、后）

测试：执行时机
static块赋值 -- 类加载
构造函数赋值 -- 类实例化
注入参数	 -- spring注入

=========================================================================================================
1、配置初始化方法init-method
执行结果的打印：
InitMethodBeanService static block execute...
InitMethodBeanService construct method execute...
InitMethodBeanService setF2 method execute...
InitMethodBeanService init method execute...
test method execute...

总结：
1、针对单个类的实例化后处理
2、执行时间是在spring容器实例化完成之后执行的

=========================================================================================================
2、实现InitializingBean接口，类实例化后处理器
执行结果的打印：
InitMethodBeanService static block execute...
InitMethodBeanService construct method execute...
InitMethodBeanService setF2 method execute...
InitMethodBeanService init method execute...
InitializingBeanService static block execute...
InitializingBeanService construct method execute...
InitializingBeanService setF2 method execute...
InitializingBeanService afterPropertiesSet method execute...
InitializingBeanService init method execute...
test method execute...

总结：
1、需要实现InitializingBean接口中的afterPropertiesSet方法
2、针对单个类的实例化后处理
3、执行时间是在spring容器实例化完成之后执行的，且在init-method方法之前
4、实现InitializingBean接口和配置init-method方法可以二者选其一，其处理的场景类似，都是对象实例化后进行对象的初始化操作

3、容器级别实例化后处理器BeanPostProcessor

执行结果的打印：
InitMethodBeanService static block execute...
InitMethodBeanService construct method execute...
InitMethodBeanService setF2 method execute...
BeanPostProcessorService postProcessBeforeInitialization method execute...
InitMethodBeanService init method execute...
BeanPostProcessorService postProcessAfterInitialization method execute...
InitializingBeanService static block execute...
InitializingBeanService construct method execute...
InitializingBeanService setF2 method execute...

InitializingBeanService init method execute...
BeanPostProcessorService postProcessAfterInitialization method execute...
BeanPostProcessorService postProcessBeforeInitialization method execute...
BeanPostProcessorService postProcessAfterInitialization method execute...
test method execute...

总结：
1、需要实现BeanPostProcessor接口中的 postProcessBeforeInitialization 方法和 postProcessAfterInitialization 方法
2、postProcessBeforeInitialization在init-method和afterPropertiesSet方法之前执行，postProcessAfterInitialization在init-method和afterPropertiesSet方法之后执行
3、postProcessBeforeInitialization 方法和 postProcessAfterInitialization 方法都是在容器实例化完成之后执行的
4、BeanPostProcessor是对容器中所有的bean执行实例化后处理
